Nuevas caracteristicas de HTML5

Comillas

Las comillas son opcionales siempre que el valor de un atrributo no contenga espacios:

ejemplos:

<section class=seccionUno>+
<section class=seccion uno>-

Doctype

Es declarado para que nuestro documento sea valido, y su nueva estructura resumida es:

<!Doctype html>

Elementos de seccionamiento

Devido al uso recurrente de ciertos nombres de identificadores(id) y clases(class), se definieron nuevos elementos de seccionamiento:

<nav> ... </nav>
<section> ... </section>
<article> ... </article>
<aside> ... </aside>

Body

El elemnto body es lo que se conoce como elemento "raiz de seccionamiento", y es importate que tenga algun encabezado para matener la estructura de la pagina:

heading para utilizar:

<h1> ... </h1>
<h2> ... </h2>
<h3> ... </h3>
<h4> ... </h4>
<h5> ... </h5>
<h6> ... </h6>

div?

Todavia se puede seguir utilizando el elemnto div si queremos separar de diferente manera cierta seccion del documento.

ejemplo:

Cuando queremos darle un estilo diferente.

<div class"estiloDiferente"> ... </div>

Contenido por defecto

Cualquier elemento que no este dentro uno de los elementos seccionadores se considera contenido por defecto o contenido implicito

MAIN

Es un nuevo elemento utilizado para el contenido principal.

  • Solo puede haber uno en el documento
  • Solo puede ser desendente de <body> ... </body> o <section> ... </section>

NAV

El elemento <nav> ... </nav> puede estar presente en en cualquier bloque que requiera navegacion.

Ejemplo: en

  • <section> ... </section>

    • y

  • <article> ... </article>

ASIDE

Utilizado para contenido adicional, generalmente que no tiene que ver con el contenido principal del documento.

Ejemplo:
  • publicidad
  • enlaces recomendados
  • ...

HEADER

header es un simplemente un contenedor que puede sostener mas de un encabezado(headings)

Ejemplo
<header>
  • <h2> Peliculas </h2>
  • <h3> Accion </h3>
</header>
Nota: Tanto header como footer pueden ser parte de cualquier elemnto seccionador.
